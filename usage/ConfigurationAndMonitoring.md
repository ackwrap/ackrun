# 配置与监控

## 配置生成

配置生成页面会根据当前数据组装完整 sing-box 配置，来源包括：

- 运行模式和代理模式。
- 订阅和启用节点。
- 节点组和策略组。
- 路由规则和规则订阅。
- DNS 和 Geo 配置。
- NTP 与实验性设置。

### 生成配置

1. 打开“配置生成”。
2. 检查“生成参数”。普通用户一般只需要确认 Mixed 端口和运行模式相关地址。
3. 点击“生成完整配置”。
4. 等待页面显示“校验通过”。
5. 展开“入站”“出站/策略组”“路由规则”“DNS”等模块检查数量和内容。

生成过程会先写临时配置，并执行 `sing-box check`。校验失败时，旧的正式配置会保留，不会被错误配置覆盖。

### 常用生成参数

- “默认出站”：通常使用 `proxy（策略）`；排查问题时可以暂时选择 `direct（直连）`。
- “日志级别”：普通使用选择 `info` 即可。
- “Mixed 监听地址”：只允许本机使用时建议 `127.0.0.1`；需要局域网设备使用时才考虑监听局域网地址，并加强 API 和防火墙保护。
- “Mixed 监听端口”：避免与其他程序冲突。
- “TUN IPv4/IPv6 地址”：使用 CIDR，并避开 LAN、Docker 和其他 VPN 已占用的网段。

### 应用配置

1. 确认生成结果显示“校验通过”。
2. 点击“应用当前生成结果”。
3. 输入配置文件名，例如 `home.json`。
4. 点击“保存并应用”。

如果没有填写 `.json`，Ackwrap 会自动补充。应用前会自动备份旧配置，并把新文件设为当前配置。核心正在运行时，应用配置会重新加载核心。

配置页面会显示每个配置文件的校验状态、当前使用标记、大小、更新时间和备份天数。

## 仪表盘

仪表盘适合在核心运行后使用。核心未运行时，页面可能提示先返回控制面板。

仪表盘可以查看：

- 实时上传和下载速率。
- 累计流量。
- 活动连接和内存使用情况。
- 策略组当前使用的节点和延迟。
- 活动连接的来源、目标、规则和代理链。
- 当前生效的路由规则。

策略组中的节点可以单独测速，也可以对整个策略组测速。测速地址来自“设置 → 常规设置 → 连通性测速”。

如果仪表盘显示“Clash API 不可用”，先确认核心是否运行，再检查“设置 → 实验性功能”中的 Clash API 端口和 API 密钥配置。

## 日志

日志页面有两个标签：

- “核心日志”：sing-box 的标准输出和错误输出。
- “工具日志”：Ackwrap 后端的操作、同步和错误信息。

可以使用“全部”“stdout”“stderr”“info”“error”筛选日志，也可以点击“刷新”。

页面默认“跟随中”，会自动滚到最新日志。需要阅读上面的旧日志时，手动滚动页面或点击“跟随”暂停自动滚动。

日志页面每类最多保留最近 1000 行。清空日志只清空当前内存中的日志，不影响订阅、节点或配置文件。

排查问题时建议按以下顺序查看：

1. 先看页面提示中的中文错误。
2. 再看工具日志中的具体操作失败原因。
3. 最后看核心日志中的 sing-box 报错。
4. 如果需要反馈问题，优先使用控制面板的“导出诊断”，不要直接上传完整日志和配置。

## 推荐验证顺序

配置生成后建议依次确认：

1. 生成结果显示“校验通过”。
2. 出站数量大于零，并且包含需要使用的节点或策略组。
3. 路由规则和 DNS 数量符合预期。
4. 应用配置后，控制面板显示配置已加载或已配置。
5. 启动核心后，日志没有持续出现错误。
6. 仪表盘可以看到核心连接和运行数据。
