# 规则与 DNS

## 规则管理

规则管理包含三部分：Geo 数据库、手动路由规则、规则订阅。

### 手动路由规则

点击“自定义规则”可以创建规则。常见规则类型包括：

- 域名：匹配一个完整域名。
- 域名后缀：匹配某个域名及其子域名。
- 域名关键词：匹配包含关键词的域名。
- IP/CIDR：匹配 IP 地址或网段。
- GeoIP/GeoSite：按地理区域或分类匹配。
- Rule Set：引用规则订阅。

每条规则需要选择出站：

- `proxy`：走代理策略。
- `direct`：直连。
- `block`：阻断。

规则从上到下匹配，越具体的规则通常越应该放在前面。点击上移或下移会立即保存排序。

系统默认的“广告拦截”和“全球直连”是受保护规则：广告拦截位于前面，全球直连作为兜底，不能随意删除或移动。

点击“查看”可以只读查看规则详情，不必进入编辑状态。点击“预览”可以查看将生成的 sing-box `route.rules` 和 `route.rule_set` JSON。

### 规则订阅

规则订阅用于订阅规则，不是代理节点订阅。支持：

- sing-box binary 规则集。
- sing-box source JSON。
- Clash Rule Provider YAML。

完整的 Clash 节点订阅应添加到订阅管理，不要添加到这里。

添加规则订阅时需要填写：

- 名称。
- URL。
- 规则集 Tag。Tag 应保持唯一、简洁，例如 `geosite-cn`。
- 格式。不了解格式时可以选择 `auto`。
- 下载方式：直连或通过代理。
- 自动同步周期。

规则订阅添加后，可以在“管理”中编辑、同步、预览转换结果、启停和删除。创建时通常可以同时生成一条引用规则，并选择它默认走代理、直连或阻断。

同步失败原因会显示在规则订阅行中。常见原因是 URL 无法访问、格式选择错误、上游返回的内容不是规则文件或规则集 Tag 冲突。

### Geo 数据库

Geo 数据库用于 GeoIP 和 GeoSite 匹配。页面默认提供 GeoIP 和 GeoSite 资源，可以单独同步或点击“同步全部”。

第一次使用 GeoIP/GeoSite 规则前，先确认对应数据库同步成功。下载失败时可以在“设置 → 检查更新”调整下载加速，或在 Geo 资源设置中确认 URL 和代理选项。

## 规则与策略的关系

可以这样理解：

1. 路由规则决定“什么流量”。
2. 出站选项决定“这些流量走哪里”。
3. 策略组决定“代理出站里面具体使用哪些节点”。

例如：创建一条“视频服务”域名后缀规则，出站选择 `proxy`，再在策略组管理中给这条代理策略配置一组节点。最终访问匹配域名时，流量才会按照这条链路处理。

## DNS 管理

DNS 管理页面分为全局设置、FakeIP、DNS 服务器和 DNS 规则。

### 添加 DNS Server

可以点击“加入内置 Server”快速添加常见的阿里、腾讯、Cloudflare、Google、Quad9 等 DNS，也可以点击“新增 Server”手动填写。

添加后注意：

- 只把自己确认可访问的 Server 保持启用。
- 全局设置中的“默认 Server”必须选择一个已启用的 Server。
- Server 的顺序可以使用上下按钮调整。
- `Detour` 表示访问该 DNS Server 时使用直连、代理或指定策略。

### 全局设置

常用选项：

- “启用 DNS 管理”。
- “默认 Server”。
- IP 返回策略：优先 IPv4、优先 IPv6、仅 IPv4 或仅 IPv6。
- 缓存容量。
- 是否禁用缓存或过期。

普通用户建议先使用默认设置，只在出现解析慢、IPv6 不通或 DNS 泄漏排查时调整。

### DNS 规则

DNS 规则从上到下匹配，适合为国内域名、局域网域名或特定查询类型指定 DNS Server。

添加 DNS 规则时需要：

1. 选择匹配条件，例如域名后缀。
2. 填写匹配值，每行一个。
3. 选择已启用的 DNS Server。
4. 按需启用“禁用缓存”和 Rewrite TTL。
5. 保存后检查规则顺序。

### FakeIP

FakeIP 由运行模式自动管理：

- TUN 和 TUN + Mixed：自动启用。
- Mixed：自动停用。

FakeIP 不是普通 DNS Server，不能像普通 Server 一样手动添加。只有确实需要避开现有局域网、Docker 或 VPN 网段时，才在配置生成参数中调整 TUN 地址或 FakeIP 网段。

## 常见关系

- 规则模式使用路由规则判断流量走向。
- DNS 规则只负责域名解析，不直接决定业务流量是否走代理。
- TUN 模式下 FakeIP 会自动参与未命中的 A/AAAA 查询。
- 修改规则或 DNS 后，建议打开配置生成页面确认校验结果。
