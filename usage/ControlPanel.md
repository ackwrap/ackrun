# 控制面板

控制面板是 Ackwrap 的首页，也是最常用的页面。侧边栏还包含仪表盘、订阅管理、节点管理、规则管理、策略组管理、DNS 管理、配置生成、日志和设置。

## 运行状态

- “未安装”：还没有安装 sing-box 核心。
- “无配置”：核心已经安装，但没有可用配置文件。
- “已停止”：核心和配置都准备好了，但进程没有运行。
- “运行中”：sing-box 正在运行。
- “异常”：核心意外退出或最近一次操作失败，需要查看错误提示和日志。

## 安装 sing-box

在“安装信息”区域点击“安装”。安装过程中会显示下载和解压进度，只有状态变为完成后才算安装成功。

如果安装失败：

1. 检查设备是否能访问 GitHub。
2. 在“设置 → 检查更新”中选择下载加速。
3. 回到控制面板重新安装。
4. 如果仍失败，打开“日志”查看工具日志，并保留错误信息。

更新运行中的 sing-box 前先停止核心，避免更新期间配置和核心状态同时变化。

## 核心控制

- “启动核心”：启动当前配置。
- “停止核心”：停止 sing-box。
- “重启核心”：停止后重新启动。
- “重载配置”：让当前运行的核心重新使用配置，实际过程会完整重启进程。

启动前 Ackwrap 会执行配置校验。配置无效时不会启动核心，也不会用坏配置覆盖原来的有效配置。

可以在核心控制区域选择当前配置文件。如果核心正在运行，切换配置后需要点击“重载配置”才会真正生效。

## 运行模式

运行模式决定流量如何进入 sing-box：

- `TUN 模式`：通过虚拟网卡接管流量，适合需要透明代理的场景。
- `Mixed 模式`：提供本机 Mixed 代理端口，浏览器或应用需要手动设置代理。
- `TUN + Mixed`：同时启用两种方式。

运行中的核心不能切换运行模式。需要切换时，先停止核心。

OpenWrt 使用 TUN 时会尝试接管本机和 LAN 流量；Mixed-only 模式不会自动接管 LAN 流量。

## 代理模式

- “规则模式”：按“规则管理”中的规则决定走代理、直连或阻断，推荐作为日常默认模式。
- “全局模式”：默认把流量交给代理策略。
- “直连模式”：默认不经过代理，适合排查问题或临时关闭代理。

代理模式也只能在核心停止时切换。切换成功后，Ackwrap 会自动生成并应用配置。

## 高级维护

- “关闭连接”：关闭当前核心管理的活动连接，不停止核心。
- “清理核心 DNS”：清理 sing-box 内部 DNS 缓存。
- “清理 FakeIP”：清理 FakeIP 映射。
- “网络自检”：检查核心程序、活动配置、核心进程、Clash API 端口和系统权限。
- “清理系统 DNS”：清理操作系统或 OpenWrt dnsmasq 的缓存。
- “清理核心日志”：清空内存中的核心日志，操作前会二次确认。
- “导出诊断”：导出脱敏诊断报告，不包含订阅地址、配置内容或节点连接参数。
- “检查更新”：检查 sing-box 是否有新版本。
- “重置防火墙”：OpenWrt 不提供此项操作；如果页面显示该按钮，请不要用于 OpenWrt 网络排查。

“重置防火墙”是危险操作，可能影响其他软件的网络访问。一般排查问题时不应首先使用它。

## 核心定时重启

控制面板的核心控制区域支持关闭、每天或每周定时重启。任务只处理正在运行的核心，核心停止时不会自动启动它。

如果订阅同步和定时重启同时发生，Ackwrap 会协调任务，避免使用旧节点状态生成配置。

## 推荐操作顺序

1. 安装 sing-box。
2. 生成默认配置。
3. 添加并同步订阅。
4. 在配置生成页面生成和应用完整配置。
5. 选择运行模式和代理模式。
6. 启动核心。
