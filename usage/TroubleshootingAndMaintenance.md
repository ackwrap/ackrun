# 故障排查与维护

## 常见问题

### 页面显示“未安装”

这是指 sing-box 核心还没有安装，不是 Ackwrap Web 页面没有安装。打开控制面板，点击“安装 sing-box”，等待安装状态变为完成。

### 页面显示“无配置”

打开控制面板点击“生成默认配置”，或者在配置生成页面生成并应用完整配置。

### 订阅同步失败

按以下顺序检查：

1. 订阅 URL 是否还能在浏览器或其他客户端访问。
2. User-Agent 是否符合订阅服务商要求。
3. 同步超时是否太短。
4. 订阅返回的内容是否确实包含节点。
5. 节点过滤规则是否把所有节点都过滤掉。
6. 是否能访问订阅服务的 HTTPS 证书和网络。

同步失败时旧节点会保留。修正设置后再次点击同步即可。

### 有节点但全部不可用

检查：

- 节点是否被禁用。
- 节点测速是否只是 TCP 可达，而协议认证本身失败。
- Hysteria、TUIC 等 UDP/QUIC 节点是否已经生成到活动配置。
- 核心是否正在运行。
- 当前模式是否误选为“直连模式”。
- 配置生成和 sing-box 校验是否通过。

### 配置校验失败

不要直接删除当前配置。先：

1. 阅读页面中的校验错误。
2. 打开日志查看工具日志。
3. 检查最近新增的节点、规则、DNS Server 或策略组。
4. 临时禁用最近新增的内容后重新生成。
5. 必要时在控制面板的当前配置下拉框中切换到之前校验通过的配置，再重载核心。

Ackwrap 的配置流程会先写临时文件并校验，校验失败时原配置通常仍然可用。

### TUN 模式无法启动

检查：

- Ackwrap 和 sing-box 是否以 root 或具备相应网络能力的用户运行。
- OpenWrt 是否提供 TUN、路由和 nftables 权限。
- OpenWrt 是否安装匹配内核的 NFQUEUE 模块。
- TUN IPv4/IPv6 网段是否和 LAN、Docker、其他 VPN 重叠。
- 是否有其他程序占用了相关路由或防火墙规则。
- 控制面板的“网络自检”结果。

如果只想让浏览器使用代理，可以先切换到 Mixed 模式，填写浏览器的 HTTP 或 SOCKS5 代理地址，暂时绕开 TUN 排查。

### Mixed 模式下浏览器不能上网

确认浏览器代理设置指向：

- 地址：运行 Ackwrap 的设备地址，单机使用时通常是 `127.0.0.1`。
- 端口：以控制面板运行统计显示的 Mixed 端口为准，不要盲目使用默认值。

如果 HTTP 代理不工作，可以同时尝试 SOCKS5 代理。修改配置生成参数或端口后，需要重新生成并应用配置。

### 仪表盘没有数据

仪表盘依赖正在运行的 sing-box 和 Clash API。先启动核心，再检查“设置 → 实验性功能”的 Clash API 端口、API 密钥和连接状态。

### 页面一直显示“连接中”

Ackwrap 使用 WebSocket 推送安装进度、订阅同步、核心状态和日志。可以依次检查：

1. 刷新页面。
2. 确认反向代理配置支持 WebSocket Upgrade。
3. 确认 API Token 已验证。
4. 查看浏览器所在网络是否拦截 WebSocket。
5. 通过页面刷新后的 REST 状态确认任务是否真的完成。

WebSocket 暂时断开时，部分页面会使用轮询补偿，不能只根据“连接中”判断任务失败。

### Chrome 开启 TUN 后访问异常

如果 Chrome 最新版本开启 TUN 后出现网页打不开、图片加载异常，而 Edge 等浏览器正常，可以尝试：

1. 在 Chrome 地址栏打开 `chrome://flags/#local-network-access-check`。
2. 将该选项设置为 `Disabled`。
3. 完全重启 Chrome。

更多背景说明见 [FakeIP 问题说明](qa/fakeip.md)。

## 备份

最重要的是备份：

- 数据库：`ackwrap.db`。
- 配置目录：`config/`。

可选备份：

- `rules/`：规则缓存，可以重新同步。
- `geo/`：Geo 数据库，可以重新下载。
- `downloads/`：核心安装包缓存，可以重新下载。

OpenWrt 默认位置示例：

```text
/etc/ackwrap/ackwrap.db
/etc/ackwrap/config/
```

## 恢复

恢复数据库或配置前先停止 Ackwrap 服务和 sing-box，避免文件被同时写入。恢复后重新启动 Ackwrap，再检查当前配置是否校验通过。

## 升级

升级前建议：

1. 备份 `/etc/ackwrap/ackwrap.db` 和 `/etc/ackwrap/config/`。
2. 记录当前核心版本、运行模式和代理模式。
3. 通过 LuCI 或 `opkg` 更新 Ackwrap IPK；更新前建议停止核心。
4. 更新后打开控制面板确认 Ackwrap 和 sing-box 状态。
5. 打开配置生成页面重新检查配置，并查看日志中是否有迁移错误。

## 浏览器代理的最简方案

如果只是想先让浏览器使用代理，可以采用下面的方案：

1. 安装 Ackwrap。
2. 安装 sing-box。
3. 生成默认配置。
4. 添加订阅并等待同步完成。
5. 到节点管理中测速，保留几个可用节点。
6. 到配置生成中生成并应用配置。
7. 控制面板选择“Mixed 模式”和“规则模式”。
8. 启动核心。
9. 在浏览器中填写 Mixed 代理地址和端口。

确认 Mixed 模式稳定后，再尝试 TUN 或 TUN + Mixed。这样出现问题时更容易判断是订阅、配置、核心还是系统路由的问题。

## 安全提醒

- 不要分享完整订阅链接。
- 不要分享包含 UUID、密码、私钥、Reality 参数或服务器地址的节点内容。
- 不要分享 API Token、Clash API 密钥或外部控制面板完整地址。
- 不要把管理端口和代理端口暴露到公网。
- 远程访问优先使用 VPN、SSH 隧道或 HTTPS。
- 配置修改前保留备份。
- 不确定某个按钮作用时，先查看页面提示和日志，不要连续重复点击危险操作。
