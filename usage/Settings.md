# 设置

设置页面包含常规设置、流量排除、实验性功能和检查更新四个标签。

## 常规设置

常规设置包含：

- 连通性测速地址。
- 节点组和策略组 URLTest 使用的测速间隔。
- GeoIP Provider。
- NTP 时间同步。

### 连通性测速

可以添加、编辑和停用 HTTP/HTTPS 测速地址，并设置全局测速间隔。该设置同时影响：

- 节点组和策略组自动 URLTest。
- 后台健康检查。
- 仪表盘策略组测速。

建议保留至少一个稳定、可访问的地址。地址必须是 HTTP 或 HTTPS URL。

### GeoIP Provider

GeoIP Provider 用于出口 IP 和路由追踪的归属地信息。普通用户可以直接使用内置 Provider，不需要自己添加。

自定义 Provider 时需要填写 HTTPS URL 和 JSON 字段映射。自定义接口由后端直连请求，不经过当前代理节点。不要添加不信任的接口地址，也不要把带有密钥的接口 URL 分享给别人。

### NTP 时间同步

NTP 用于保持 sing-box 内部时间准确。Reality、VLESS-XTLS 和 TLS 校验场景建议开启 NTP。

修改 NTP 后，提示“下次生成配置时生效”是正常的。常见设置包括 NTP 服务器、端口、同步间隔和出站策略。

## 流量排除

流量排除用于让某些进程、网络接口、IP/CIDR 或域名后缀绕过透明代理，优先直连。

可选类型包括：

- 进程名称。
- 网络接口。
- 目标 IP/CIDR。
- 来源 IP/CIDR。
- 域名后缀。

常见用途是排除局域网网段、某个虚拟网卡或 EasyTier 等网络工具。添加后点击“保存流量排除设置”，配置会自动更新。

不要随意排除大范围网段，否则可能造成流量绕过代理或 DNS 规则。

## 实验性功能

实验性功能包括 Clash API、外部控制面板和缓存文件。

### Clash API 和外部控制面板

可以安装和使用 MetaCubeXD、Yacd-meta 或 Zashboard 等外部控制面板。操作顺序是：

1. 在下拉框选择控制面板。
2. 如果显示未安装，点击“安装面板”。
3. 设置 Clash API 端口和 API 密钥，建议点击“随机生成”。
4. 保存设置并等待配置应用。
5. 使用“打开面板”进入外部控制面板。

外部面板地址可能包含连接参数和 API 密钥。它等同于凭据，不要发到公开群聊、截图或工单中。

### 缓存文件

缓存文件可以保存 FakeIP 映射和完整 DNS 缓存，减少重启后的重复解析。普通用户可以保持默认启用。

## 检查更新

这里可以检查 Ackwrap WebUI 和后端的最新版本，设置下载加速，并在发现新版本后执行更新。

更新前建议：

- 先备份数据库和配置目录。
- 确认没有正在进行的重要同步任务。
- 运行中的核心通常会在更新后自动恢复，但仍建议提前记录当前运行模式。

下载加速支持直连、GitHub 加速和自定义镜像。启用自定义镜像后，请确认镜像可信且可访问。
