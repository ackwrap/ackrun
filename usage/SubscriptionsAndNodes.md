# 订阅与节点

## 订阅管理

订阅管理负责保存外部订阅链接，并把订阅解析成节点。

### 添加订阅

点击右上角的“+”，填写：

- 名称：给自己看的备注，例如“家庭订阅”。
- 订阅链接：通常以 `https://` 开头。
- User-Agent：优先使用订阅服务商建议的选项，不确定时可以先使用默认值。
- 自动同步：可选择关闭、每天、每周或每月。
- 同步时间：自动同步的时间。
- 同步超时：网络较慢时可以适当调大。

Ackwrap 支持以下订阅内容：

- sing-box JSON。
- Clash YAML。
- Base64 编码的 URI 列表。
- 普通 URI 列表。

保存订阅后通常会自动同步一次，不需要马上点击第二次同步。

### 同步订阅

- 单个订阅：点击该行的刷新按钮。
- 全部订阅：点击“同步所有订阅”。
- 查看节点：点击订阅行中的节点数，跳转到对应节点列表。

同步状态通常包括“同步中”“已更新”和“失败”。同步失败时，错误原因会显示在订阅行下方或提示消息中。

同步失败不会清空旧节点。只有解析成功并且过滤后仍有节点时，才会替换该订阅的节点。

### 手动导入节点

订阅页面下方的“本地订阅导入”用于导入临时节点或单个节点。支持：

- URI List：每行一个节点 URI。
- Base64 URI List。
- Clash YAML。
- sing-box JSON。

操作步骤：

1. 把节点内容粘贴到文本框。
2. 点击“节点预览”，确认识别出的数量和节点名称。
3. 点击预览列表中的节点，可以查看 JSON 或 YAML 详情。
4. 确认无误后点击“导入节点”。

手动导入的节点会保存到“本地节点源”，不会清空以前导入的节点；相同 UID 的节点会追加或更新。

不要把包含密码、UUID、私钥或完整连接参数的节点内容粘贴到公开聊天、工单或截图中。

## 节点过滤规则

节点过滤规则位于订阅页面下方。它不是页面临时搜索，而是后端在订阅同步或手动导入时应用的规则。

可以按这些字段过滤：

- 全部字段。
- 节点名称。
- 协议类型。
- 服务器地址。
- 原始内容。
- 解析后的 JSON。

当前规则的含义是：节点命中启用的正则后，会被排除。例如：

- 排除名称中包含“过期”的节点：目标字段选择“节点名称”，正则填写 `过期`。
- 排除某种协议：目标字段选择“协议类型”，正则填写对应协议名称。

如果只想使用某个地区的节点，不能直接填写该地区名称，因为这样会把命中的节点排除；可以在节点管理中临时筛选，或在策略组管理中创建包含关键词的节点组。

过滤后一个节点都没有时，同步会失败并保留旧节点。因此新增复杂正则后，建议先手动同步一个订阅确认结果。

## 节点管理

### 筛选和搜索

页面提供：

- 按协议筛选。
- 按订阅筛选。
- 按节点状态筛选。
- 按启用状态筛选。
- 按首选状态筛选。
- 搜索名称、地址或 UID。

筛选器上的数量来自后端统计，不是当前页面剩下的数量。切换筛选条件后，列表会重新加载。

### 启用、禁用和首选

- “启用”：节点会参与后续配置生成。
- “禁用”：节点保留在数据库中，但不会参与新的配置生成。
- “首选”：只是给自己做标记，方便筛选和管理，不等于强制使用该节点。

建议优先使用“禁用”而不是删除节点。订阅下次同步时，被删除的节点可能再次出现，而禁用状态可以按稳定 UID 继承。

### 节点测速

点击单个节点的延迟数字可以测试该节点，也可以先勾选多个节点，再点击“节点测速”。

测速结果只代表连通性：

- 普通 TCP 节点测试 TCP 连接耗时。
- Hysteria、Hysteria2、TUIC、WireGuard 等 UDP/QUIC 节点使用运行中的 sing-box 真实出站测试。
- 测速不会验证订阅密码、UUID、TLS 或 Reality 参数是否完全正确。
- 测速不会代表实际下载速度。

如果 UDP/QUIC 节点提示未载入活动配置，先生成并应用配置，再启动核心后重试。

### 批量操作

勾选节点后可以：

- “添加 emoji”：根据节点名称推断国家或地区并添加国旗。
- “修改名称”：支持添加前缀、添加后缀、查找替换和按行改名。
- “管理首选”：批量标记首选。
- “批量启用”或“批量禁用”。
- “批量删除”：不可恢复，使用前确认已经不需要这些节点。

改名和添加 emoji 后会保护自定义名称，后续订阅同步不会覆盖它。

### 路由追踪和出口 IP

- “路由追踪”：查看 Ackwrap 主机到节点服务器之间经过的网络跳数，不是代理节点出口的路径。
- “出口 IP”：尝试通过当前节点访问检测服务，查看入口 IP 和出口 IP。

出口 IP 检测需要核心运行、节点已启用并且节点已经载入活动配置。入口 IP 与出口 IP 不同不一定是故障，中转、NAT 和 Anycast 都可能导致它们不同。
