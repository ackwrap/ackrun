# 策略组

策略组管理页面分为两个标签：

- “节点组（地域划分）”：把节点按条件归类。
- “策略组（业务用途）”：把节点组用于代理、自动测速或手动选择。

## 节点组

节点组可以按以下条件筛选节点：

- 协议限制。
- 订阅限制。
- 包含关键词。
- 排除关键词。

点击“详情”可以查看实际匹配的节点。节点发生变化时，动态节点组也会跟着更新。

如果不想手动逐个创建，可以点击“智能快速配置”，选择订阅和协议。全部不选或全部选中表示不限制，包含以后新增的订阅或协议。

## 策略组

策略组与规则管理中的业务规则对应。常见类型：

- `selector`：手动切换节点。
- `urltest`：按连通性测速自动选择较快的节点。
- 直连和拒绝策略：系统固定策略，不需要手动配置节点。

对“代理”类型的业务规则点击“配置”，然后选择节点来源：

- 引用节点组。
- 引用节点组和组内实际节点。
- 手动选择节点。

URLTest 使用“设置 → 常规设置 → 连通性测速”中的地址和间隔。建议先在常规设置中保留至少一个可访问的 HTTP/HTTPS 测速地址。

策略组的顺序跟随路由规则，不能在此页面单独排序。需要调整先后时，请去“规则管理”中移动规则。

## 与节点的关系

可以这样理解：

1. 订阅同步产生节点。
2. 节点组根据条件筛选节点。
3. 策略组引用节点组或手动选择的节点。
4. 路由规则决定哪些流量使用这个策略组。

例如，可以创建一个包含“日本”的节点组，再为视频类代理规则配置一个 URLTest 策略组。这样匹配该规则的流量会在这批节点中自动选择较快的节点。

## 配置建议

- 节点较少时，可以直接手动选择节点。
- 节点经常变化时，优先使用动态节点组。
- 需要自动切换时选择 URLTest。
- 需要自己决定当前节点时选择 Selector。
- 先确认节点组详情中有匹配节点，再保存代理策略。
