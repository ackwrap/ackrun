# 后期计划

## 代理集合

参考 Clash Meta / Mihomo 的 `proxy-providers` 概念，但不直接照搬命名和配置结构。

### 背景说明

Clash Meta 的 `proxy-providers` 用于定义一组可从外部订阅源加载的代理节点集合，例如：

- 从订阅链接下载一组节点
- 给这组节点命名
- 设置自动更新周期
- 设置健康检查 / 测速地址
- 在策略组中引用该节点集合

在 AckWrap 中，目标是管理 sing-box，因此后续应抽象为更通用的“代理集合”，而不是直接使用 Clash 的 `proxy-providers` 名称。

### 建议产品概念

后续页面可以拆分为：

- 订阅管理：管理外部订阅源 URL
- 节点管理：展示和管理订阅解析后的具体节点
- 代理集合：管理一组节点的分组和使用策略

### 代理集合职责

代理集合用于：

- 将多个节点组合成一个集合
- 设置集合名称，例如“机场 A”“香港节点”“自动测速组”
- 选择集合包含哪些订阅源或具体节点
- 设置自动更新策略
- 设置健康检查 / 测速 URL
- 后续生成 sing-box 配置时，映射为 selector / urltest 等 outbounds

### 开发顺序建议

1. 完成订阅管理 CRUD
2. 实现订阅同步与节点解析
3. 完成节点管理页面
4. 增加代理集合页面
5. 支持从代理集合生成 sing-box 配置
6. 支持健康检查、测速、自动选择策略

### 注意事项

- 不要直接把 Clash Meta 配置结构暴露给用户
- UI 层使用“代理集合”这类产品化名称
- 内部可以根据需要适配 sing-box 的 outbound / selector / urltest 配置
- 第一版不要做复杂策略编排，先保证订阅、节点、集合三层关系清晰
